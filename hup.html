The Hup scripting language

Julien Laguerre

This is a stack.

1 2 3

And this is how you add 1 to each item of this stack.

1 2 3 + 1

The result of this expression is another stack.

2 3 4

There is no operator precedence. Instead, an expression is read from left to right. Values are pushed on an operand stack. Operators are applied to the operand stack, resulting in a new stack.

1 2 3 + 1 * 2

Here, 1 is first added to each item of the 1 2 3 stack, and then each item of the resulting stack is doubled, which results in:

4 6 8

Now this is a function call. The name of the function is "a", and it has 2 arguments "b" and "c".

a(b c)

When an expression is given as argument to a function, the expression is first resolved before the function gets called on the resulting stack.

f(1 2 3 + 1 * 2)

This is equivalent to:

f(4 6 8)

Now, this is a table.

[ ]

A table may have a name, a formula, and values obtained by executing the formula, always in that order. The formula is preceded by equal "=" and each value is preceded by a vertival bar "|".

[ example = 1 2 3 + 1 * 2 | 4 | 6 | 8 ]

White space is irrelevant.

[
    example
    = 1 2 3 + 1 * 2
    | 4
    | 6
    | 8
]

The formula is optional. A table without formula is a constant table.

[
    some numbers
    | 1
    | 2
    | 3
]

The formula of a table can reference the values of other tables. The name of the referenced tables are then put between square brackets.

[
    example
    = [some numbers] + 1 * 2
    | 4
    | 6
    | 8
]

When the values of a referenced table change, the values of the referencing tables are recalculated to reflect the changes. If [some numbers] changes to 10 20 30, then [example] will automatically change too.

[ some numbers | 10 | 20 | 30 ]
[ example = [some numbers] + 1 * 2 | 11 | 22 | 33 ]

It is thus an event driven system.
